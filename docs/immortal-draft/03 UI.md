# Immortal Draft - UI Discussion

This document covers the UI/UX design for the draft system in Discord.

---

## UI Components Needed

1. **First/Second Pick Choice** - Lower-rated captain chooses pick order
2. **Player Selection** - Captains pick players during draft
3. **Draft Status Display** - Shows current state, teams, available players

---

## Discord UI Constraints

| Component | Limit |
|-----------|-------|
| Buttons per row | 5 |
| Button rows per message | 5 |
| Total buttons per message | 25 |
| Dropdown options | 25 |
| Embed fields | 25 |
| Embed field name | 256 chars |
| Embed field value | 1024 chars |
| Message content | 2000 chars |

For our use case (8 available players), all options work within limits.

---

## Part 1: Pre-Draft Setup (Multi-Step)

Before the player draft begins, there are several choices to make:

### Two Separate Drafts

| Draft | What | Timing |
|-------|------|--------|
| **Player Draft** | Captains pick players for teams (1-2-2-2-1) | Before match |
| **Hero Draft** | Players pick heroes in-game | During match |

### Pre-Draft Flow

```

Step 2: Coinflip winner chooses ONE of:

        â€¢ 1st/2nd pick in hero draft

Step 3: Other captain chooses the remaining option

Step 4: Lower-rated captain chooses player draft order (1st/2nd)

Step 5: Player draft begins
```

- Message needs to be re-sent or edited on each pick

**Implementation Notes:**
- Use `discord.ui.Button` in a `discord.ui.View`
- Validate that clicker is the current captain
- Edit message after each pick to update state

## My Recommendation

### Primary Choice: Buttons (Option A)

**Reasons:**
1. Most intuitive for captain mode drafts
2. Fast interaction (single click)
3. Visual - see all options at once
4. Good mobile experience
5. Easy to validate (only current captain's click counts)
6. 8 players fits perfectly in 2 rows

### Implementation Approach

1. **Single updating message** - Edit the same message after each pick
2. **Embed for status** - Teams, turn info, available players
3. **View for buttons** - Player buttons below the embed
4. **Timeout on View** - Buttons persist until draft complete or cancelled

### Code Structure

```python
class DraftView(discord.ui.View):
    def __init__(self, draft_state, timeout=None):
        super().__init__(timeout=timeout)
        self.draft_state = draft_state
        self._add_player_buttons()

    def _add_player_buttons(self):
        for player in self.draft_state.available_players:
            button = PlayerPickButton(player)
            self.add_item(button)
```

```python
class SidePreferenceButton(discord.ui.Button):
    async def callback(self, interaction):
        user_id = interaction.user.id

        # Only available players can set preference
        if user_id not in draft_state.available_player_ids:
            await interaction.response.send_message(
                "Only available players can set preferences!",
                ephemeral=True
            )
            return

        # Update preference
        draft_state.set_side_preference(user_id, self.side)  # "radiant" or "dire"

        # Refresh the draft embed
        await interaction.response.edit_message(
            embed=build_draft_embed(draft_state),
            view=build_draft_view(draft_state)
        )
```
